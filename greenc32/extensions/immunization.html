3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Immunization contains information about the immunization history for a patient.

4.2. Description

NameTypeMandatory/OptionalNotes
id string M Entry Component: Unique id for the immunization entry
medicationCode code M Entry Component: code - Coded value of the administered immunization
medicationName string M Entry Component: freeText - Name of the administered immunization
administeredDate interval O Entry Component: Effective Date - The date and time of substance was administered or refused.
medicationSeriesNumber integer O Indicates which in a series of administrations a particular administration represents.
refused boolean M A flag that the immunization event did not occur.
refusalReason Code O

When an immunization is refused, this provides a coded representation of the reason for refusing the immunization

  • IMMUNE -- Immunity
  • MEDPREC -- Medical precaution
  • OSTOCK -- Out of Stock
  • PATOBJ -- Patient objection
  • PHILISOP -- Philosophical objection
  • RELIG -- Religious objection
  • VACEFF -- Vaccine efficacy concerns
  • VACSAF -- Vaccine safety concerns
performingProvider URL O Link for the person that administered the immunization to the patient
reaction URL O

Link to Allergy entry for noted intended or unintended effects of the product. For example: full body rash, nausea, rash resolved

4.3. Clinical Requirement

Immunization is characterized by:

  • refusal of Immunization
  • immunization product
  • noted reactions

Immunizations may be represented in XML and JSON format.
