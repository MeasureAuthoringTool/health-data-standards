3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Encounter contains information defining the entrie describing encounters for a patient.

4.2. Description

NameTypeMandatory/OptionalNotes
id string M Entry Component: Unique id for the encounter
code Code M Entry Component: The coded value describing the type of encounter
status string O

Entry Component: The status of the encounter.

freeText string O Entry Component: Free text describing the Encounter
effectiveTime timeInterval M

Entry Component: The date and time of the Encounter, including duration if pertinent
dischargeDisposition string O

The patient's anticipated location or status following the encounter – uses standard claims-based codes
admissionType SimpleCode O

A code indicating the priority of the admission (e.g., Emergency, Urgent, Elective, etc)

  • 1 -- Emergency
  • 2 -- Urgent
  • 3 -- Elective
  • 4 -- Newborn
  • 5 -- Trauma Center
  • 9 -- Information Not Available
facility organizationInformation O

The service delivery location
facilityDuration timeInterval O The duration for which a patient is at a specific location as calculated from the admission/registration and discharge dates.
reasonForVisit codedValue O Indicates the rationale for the encounter. Equivalent to admitting diagnosis and contains condition code.
performer URL M

Link to Provider entry for the person or organization performed or hosted the Encounter

4.3. Clinical Requirement

Encounter is characterized by:

  • Rationale for the encounter;
  • Text describing the encounter;
  • Location of service and duration;
  • Healthcare provider; and
  • Encounter identifier, status, date, location, duration.

Encounter may be represented in XML and JSON format.
