3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Medication contains information defining the entry describing relevant medications associated with a patient. It can include active and non-active, prescription and non-prescription items. 0 to many medications can be associated with a referralConsultRequest . Medication is also includes the following subclasses: FulfillmentHistory, OrderInformation, and MedicationInformation.

4.2. Description

The instructions, typically from the ordering provider, to the patient on the proper means and timing for the use of the product. This information is free-text but can also be represented as a series of Sig Components
NameTypeMandatory/OptionalNotes
id string M Entry Component: Unique id for the medication
code Code M Entry Component: The coded value of the medication represented
status simplecode O

Entry Component: The status of the medication. If the medication is Active, Discharged, Chronic, Acute, etc

administrative timing period quantity O A Sig Component: defines a specific administration or use time. Can be a text string (Morning, Evening, Before Meals, 1 Hour After Meals, 3 Hours After Meals, Before Bed) or an exact time
administrative timing institutionalSpecified boolean O Indicates if the administration is institutional specific
route Code O A Sig Component: indicates how the medication is received by the patient (e.g., by mouth, intravenously, topically, et cetera)
dose quantity O A Sig Component: the amount of the product to be given. This may be a known, measurable unit (e.g., milliliters), an administration unit (e.g., tablet), or an amount of active ingredient (e.g., 250 mg). May define a variable dose, dose range or dose options based upon identified criteria.
doseIndicator string O A Sig Component: A criteria that specifies when an action is, or is not, to be taken. For example, "if blood sugar is above 250 mg/dl"
site Code O A Sig Component: The anatomic site where the medication is administered. Usually applicable to injected or topical products.
doseRestriction quantity O A Sig Component: defines a maximum or dose limit. This segment can repeat for more than one dose restriction.
productForm Code O The physical form of the product as presented to the patient. For example: tablet, capsule, liquid or ointment
deliveryMethod code O A Sig Component: A description of how the product is administered/consumed
type code O

This is a classification based on how the medication is marketed (e.g., prescription, over the counter drug) Constrained to:

  • 329505003 -- Over the counter products(product)
  • 73639000 -- Prescription drug(product)
status of medication Code O

indication Code O

A Sig Component: The medical condition or problem intended to be addressed by the ordered product. For example: for chest pain, for pain, for high blood pressure. Describes the problem. The SNOMED CT® has been limited by HITSP to terms decending from the Clinical Findings (404684003) or Situation with Explicit Context (243796009) hierarchies.
vehicle Code O

Shall contain a concept descending from code 412307009 from the SNOMED CT®

A Sig Component: Non-active ingredient(s), or substances not of therapeutic interest, in which the active ingredients are dispersed. Most often applied to liquid products where the major fluid Component is considered the vehicle.

For example: Normal Saline is the vehicle in "Ampicillin 150mg in 50ml NS"; Aquaphor is the vehicle in "10% LCD in Aquaphor"

patientInstructions string O Instructions to the patient that are not traditionally part of the Sig. For example, "keep in the refrigerator." More extensive patient education materials can also be included
fulfillmentInstructions string O Instructions to the dispensing pharmacist or nurse that are not traditionally part of the Sig. For example, "instruct patient on the use of occlusive dressing"
prescriptionNumber string O Fulfillment History component:The prescription identifier assigned by the pharmacy
dispenseDate DateTime O Fulfillment History component:The date of this dispense
quantityDispensed Quantity O Fulfillment History component:The actual quantity of product supplied in this dispense. Note: this is comprised of both a numeric value and a unit of measure
fillStatus String O Fulfillment History component:This identifies whether the medication has been fulfilled, such as completed and aborted
quantity Ordered Quantity O Order Information component:The amount of product indicated by the ordering provider to be dispensed. For example, number of dosage units or volume of a liquid substance. Note: this is comprised of both a numeric value and a unit of measure
ordered date time DateTime O Order Information component:The date, including time if available, when the ordering provider wrote the order/prescription
expiration date time DateTime O Order Information component:The date, including time if applicable, when the order is no longer valid. Dispenses and administrations are not continued past this date for an order instance
orderNumber String O Order Information component:The order identifier from the perspective of the ordering clinician. Also known as the 'placer number' versus the pharmacies prescription number (or 'filler number')
fills Integer O Order Information component:The number of times that the ordering provider has authorized the pharmacy to dispense this medication
dispensingPharmacy URL O Fulfillment History component:Link to the provider entry for the pharmacy that performed this dispense
orderingProvider URL O Order Informaiton component:Link to Healthcare Provider entry for the provider who ordered the medication
